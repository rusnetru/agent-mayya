# AGENTS.md — инструкция для AI-агентов, работающих с проектом

## Структура

- `src/` — исходный код
- `docs/` — документация
- `tests/` — 146 тестов (pytest)
- `evals/` — сценарии качества (`python -m evals.run`)
- `skills/` — навыки Mayya (SKILL.md, система как у Hermes)
- `mcp.json` — MCP-серверы (playwright браузер, github); клиент в `src/mcp/client.py`

## Ключевые документы

| Файл | Для чего |
|------|----------|
| `soul.md` | **Портрет пользователя и правила безопасности — читать при старте** |
| `docs/00_progress_log.md` | История разработки по фазам, текущий статус |
| `docs/05_pipeline_and_roadmap.md` | Карта модулей и план дальнейшей разработки |
| `docs/04_deployment.md` | Установка, обновление, запуск |
| `docs/06_telegram_setup.md` | **Инструкция по настройке Telegram-бота** |

## Настройка Telegram

Если пользователь просит «настрой Telegram», «подключи бота» или передаёт токен:

1. Прочитай `docs/06_telegram_setup.md`
2. Выполни шаги 1-7 из раздела 2
3. Сообщи пользователю результат

Пользователю нужно только передать токен от @BotFather — всё остальное делает агент.

## Запуск

```bash
.venv/Scripts/python -m pytest -q   # тесты
python src/main.py                   # CLI-режим
python src/main.py --telegram        # Telegram-режим (после настройки)
```

## Правила

- Не менять системные настройки Windows
- Токены и ключи — в `.env`, не в коде
- После правок — `pytest`
- Документировать фазы в `docs/00_progress_log.md`
