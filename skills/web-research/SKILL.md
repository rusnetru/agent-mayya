# Web Research — как искать глубоко

Для простого фактического вопроса достаточно одного web_search. Этот навык — для задач
«разберись», «сделай обзор», «сравни», «подбери» — где один запрос даёт поверхностный ответ.

## Алгоритм

1. **Сформулируй 2–3 разных запроса**, а не один:
   - основной запрос на языке пользователя;
   - вариант на английском (в англоязычном интернете больше материала);
   - уточнённый вариант (год, «vs», «обзор», «how it works»).
2. **web_search по каждому.** Провайдер Serper может вернуть «Прямой ответ Google» первым
   результатом — используй его как быстрый ориентир, но не как единственный источник.
3. **Отбери 2–3 непохожих источника** (не три копии одной новости) и прочитай их через
   web_extract. Если страница не открылась — бери следующую, не зацикливайся.
4. **Синтезируй ответ сам**: сведи источники воедино, отметь противоречия, укажи даты.
   В конце — список использованных URL.

## Правила

- Пустой результат поиска → переформулируй запрос (синонимы, английский), попробуй ещё 1–2 раза.
- Не выдумывай факты, которых нет в источниках. Нет данных — так и скажи.
- Для свежих тем (новости, версии, цены) обязательно смотри дату материала и говори её пользователю.
