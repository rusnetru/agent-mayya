# Browser Automation — живой браузер через Playwright MCP

## Когда использовать

- Страница не отдаётся через web_extract (JS-рендеринг, SPA, логин).
- Нужно взаимодействие: клики, формы, скролл, скриншот.
- Нужно проверить, как сайт выглядит/работает на самом деле.

Для простого чтения статичных страниц web_extract быстрее — начинай с него.

## Основные инструменты (префикс playwright_)

- `playwright_browser_navigate(url)` — открыть страницу; в ответе URL, title и снапшот
- `playwright_browser_snapshot()` — accessibility-снимок страницы: текст + ref-ссылки на элементы
- `playwright_browser_click(element, ref)` — клик по элементу из снапшота (передавай и описание, и ref)
- `playwright_browser_type(element, ref, text)` — ввод текста в поле
- `playwright_browser_evaluate(function)` — выполнить JS на странице
- `playwright_browser_take_screenshot()` — скриншот
- `playwright_browser_close()` — закрыть страницу когда закончила

## Алгоритм

1. navigate → смотри снапшот из ответа.
2. Нужен элемент — бери его `ref` из снапшота, не выдумывай селекторы.
3. После каждого действия читай новый снапшот (действия возвращают его сами).
4. Закончила — browser_close, не оставляй браузер висеть.

## Правила

- Браузер headless — пользователь его не видит, описывай словами что нашла.
- Не логинься на сайты и не вводи персональные данные без явной просьбы пользователя.
- Если MCP-сервер playwright не подключён (нет playwright_* инструментов) — скажи об этом и используй web_extract.
