# Отчёт по разработке Mayya — 3 июля 2026 (Claude Fable 5)

Продолжение работы после отчёта Hermes (`development-report-2026-07-03.md`).

## Диагноз: почему Mayya была «деревянной»

Главная причина — архитектурная. Каждое сообщение пользователя шло не через диалог, а через конвейер задач:

```
сообщение → decompose (режет фразу по «и»!) → LLM-маршрутизация фрагмента
  → subagent видит только фрагмент без истории → verifier (ещё один LLM-вызов)
  → при FAIL весь цикл повторяется до 3 раз → из транскрипта берётся одна строка
```

Следствия: потеря контекста, переспрашивания, медлительность, пассивность. Память из 4 уровней существовала, но в диалог не подключалась вообще — Mayya ничего не «помнила» на практике.

## Найденные баги

1. **Пустой DuckDuckGo — это был баг парсера, не ограничение DDG.** Ссылки в выдаче — redirect-URL `//duckduckgo.com/l/?uddg=<encoded>`, а фильтр `url.startswith("http")` отбрасывал их все.
2. **Tool-схемы**: все параметры объявлены строками и обязательными. DeepSeek передавал `max_results="3"` → `TypeError` на срезе списка.
3. **HTML-сущности** в web_extract fallback (`&#x2F;` в тексте).

## Что сделано

### Новое ядро: ConversationalAgent (`src/agent/conversational.py`)

Живой agent loop — основной путь диалога:
- настоящая история сообщений (messages array, не текстовый блоб)
- все 6 инструментов, до 12 итераций tool calling за ход
- recall эпизодической памяти в system prompt перед каждым ходом
- диалог сохраняется в память → Mayya помнит между сессиями
- при исчерпании бюджета шагов — принудительный итоговый ответ из найденного
- история хранит только user/assistant-текст (нет осиротевших tool_call id), обрезается до 30 сообщений

Оркестратор с сабагентами не удалён — доступен явно через `/task <задача>`.

### Починки

- `web_search.py`: декодирование uddg-редиректов, POST html.duckduckgo.com + fallback lite.duckduckgo.com
- `registry.py`: JSON-схемы с типами, необязательные параметры, приведение типов и фильтрация лишних аргументов
- `web_extract.py`: html.unescape в fallback
- `subagent.py`: сабагенты получают `context.history()` — executor видит результаты researcher (проблема №3 Hermes)
- `personality.py`: живой характер; «2-3 попытки при ошибке, потом честно скажи»; запрет кода-заглушек (проблема №2 Hermes)
- `main.py`: диалог через ConversationalAgent, `/task`, `/new` сбрасывает агента
