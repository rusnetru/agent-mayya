# План тестирования Mayya — функциональность AI-агента

Версия: 1.0 · 2026-07-03 · 87 тестов (unit) + ручные сценарии

## Цель

Проверить что Mayya работает как **проактивный AI-агент**: сама ищет информацию, вызывает инструменты, не переспрашивает, отвечает по делу.

## Запуск

```
cd agent-mayya
PYTHONPATH=. .venv/Scripts/python src/main.py
```

## 4. Отказы без инструментов

Mayya **НЕ должна** говорить эти фразы:

- ❌ «Я не могу переходить по ссылкам»
- ❌ «Я не имею доступа к браузеру»
- ❌ «Если хочешь, я могу поискать...»
- ❌ «Уточните, что именно нужно найти»
- ❌ «Для выполнения этого запроса мне нужен контекст»

Вместо этого — **действие**: поиск → загрузка → ответ.

## 6. Оркестратор и subagent'ы

| # | Сценарий | Ожидаемый результат |
|---|----------|---------------------|
| 6.1 | Задача с поиском (`найди X`) | Routing → researcher → web_search → результат. |
| 6.2 | Задача без поиска (`представься`) | Routing → executor → прямой ответ. |
| 6.3 | Сложная задача (`найди X и создай файл с результатом`) | Decompose → researcher ищет → executor создаёт файл. |
| 6.4 | Проверка (`проверь что файл test.txt существует`) | Routing → verifier → PASS/FAIL. |
| 6.5 | Verifier FAIL | Если задача не выполнена — verifier возвращает FAIL, видно в транскрипте. |
