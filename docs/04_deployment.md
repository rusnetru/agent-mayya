# Развёртывание Next Gen Agent из репозитория

Репозиторий: https://github.com/rusnetru/next-gen-agent

Два сценария:
1. **Разработка** — клонируете репозиторий, работаете с исходным кодом, запускаете тесты.
2. **Конечный пользователь (Windows)** — собираете один `.exe` и запускаете его без Python.

---

## 1. Требования

| Что | Версия |
|---|---|
| Python | 3.11+ |
| ОС | Windows 10/11 (основная цель сборки); Linux/macOS — для разработки тоже подходит |
| Git | любой современный |

Проверить, что Python установлен:

```powershell
python --version
```

Если команда не найдена — поставить Python с [python.org](https://www.python.org/downloads/) (отметить галку "Add python.exe to PATH" при установке).

---

## 2. Сценарий А: разворачиваем для разработки

### 2.1 Клонировать репозиторий

```powershell
git clone https://github.com/rusnetru/next-gen-agent.git
cd next-gen-agent
```

### 2.2 Создать виртуальное окружение и поставить зависимости

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

Если PowerShell блокирует активацию скрипта (`...cannot be loaded because running scripts is disabled...`), один раз разрешить для текущего пользователя:

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

### 2.3 Проверить, что всё работает — прогнать тесты

```powershell
python -m pytest -q
```

Ожидаемый результат: все тесты зелёные (на момент Фазы 5 — 70 passed).

### 2.4 Запустить агента напрямую (без сборки в exe)

```powershell
python src/main.py
```

Это вызывает Phase 0 agent loop (`src/agent/loop.py`) — минимальный пример. Рядом с местом запуска появится файл `memory.db` — персистентная episodic-память (SQLite), переживает перезапуск процесса.

### 2.5 Конфигурация (LLM backend)

Для реальных вызовов Claude API (а не текущих детерминированных заглушек субагентов) понадобится переменная окружения с ключом:

```powershell
$env:ANTHROPIC_API_KEY = "sk-ant-..."
```

Или создать `.env` в корне репозитория (он уже в `.gitignore`, не попадёт в git):

```
ANTHROPIC_API_KEY=sk-ant-...
```

> На текущий момент (Фаза 5) субагенты (`src/orchestrator/subagents.py`) — это детерминированные заглушки без сетевых вызовов, ключ API пока ничего не использует. Подключение реального вызова Claude внутрь `Subagent.act()` — следующий шаг по плану (см. [docs/00_progress_log.md](00_progress_log.md), раздел «Следующий шаг»).

## 3. Сценарий Б: собрать Windows .exe и развернуть на машине без Python

### 3.1 Собрать

В уже подготовленном окружении разработки (шаги 2.1–2.2 выше):

```powershell
.\build.ps1
```

Это эквивалент:

```powershell
pyinstaller --noconfirm --onefile --name next-gen-agent src/main.py
```

Результат — `dist\next-gen-agent.exe` (однофайловый, ничего больше копировать не нужно).

> Сборка занимает заметное время (несколько минут) и даёт файл ~230 МБ — PyInstaller затягивает в бандл тяжёлые транзитивные зависимости `chromadb` (`torch`, `onnxruntime`), хотя сам код их пока не использует напрямую. Если важен компактный размер — см. техническую заметку в [docs/00_progress_log.md](00_progress_log.md).

### 3.2 Развернуть на целевой Windows-машине

Никакой установки Python не требуется — только сам `.exe`.

1. Скопировать `dist\next-gen-agent.exe` на целевую машину (флешка, сетевая папка, артефакт CI — что угодно).
2. Запустить:

```powershell
.\next-gen-agent.exe
```

3. При первом запуске рядом с `.exe` создастся `memory.db` — это персистентная память агента. При повторных запусках из той же папки агент будет помнить предыдущие эпизоды.

Если нужно «чистое» состояние — удалить `memory.db` перед запуском.

### 3.3 Проверка, что сборка рабочая

Признак успеха — в консоли появляется вывод вида:

```
[stub action] respond to: hello
```

и в папке рядом с `.exe` появляется файл `memory.db`.

## 4. Типичные проблемы

| Проблема | Решение |
|---|---|
| `python: command not found` | Установить Python, добавить в PATH, перезапустить терминал |
| PowerShell не даёт активировать `.venv` | `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` |
| `pip install` падает на `chromadb`/`torch` (нет интернета или мало места на диске) | Эти пакеты — самые тяжёлые в `requirements.txt`; нужен стабильный интернет и от 2 ГБ свободного места |
| Сборка `.exe` падает или зависает надолго | Это нормально для первого запуска (PyInstaller анализирует весь граф импортов `chromadb`); повторные сборки в той же папке (`build/` уже есть) идут быстрее |
| `.exe` не запускается на целевой машине ("missing DLL" и т.п.) | Собирать `.exe` на машине с той же разрядностью/версией Windows, что и целевая (PyInstaller не кросс-компилирует) |
| Антивирус блокирует `.exe` | Частая ложная тревога для PyInstaller-сборок без цифровой подписи — добавить файл в исключения или подписать сертификатом для продакшн-распространения |
